2. UI/UX Overhaul Needed

We identified specific areas where the visual design can be elevated to meet the premium standard we are aiming for.

App Icon

Problem: The current asset is low-resolution and lacks distinctiveness.

Solution: We will design a vector-based, high-fidelity icon that utilizes the new "Safety Teal" palette. This ensures brand recognition at any scale and communicates trust instantly.

Phone Number Input

Problem: The raw text input lacks visual validation.

Solution: Implement `libphonenumber` for real-time formatting and a visual country code selector. This reduces cognitive load and prevents invalid data entry.

Onboarding Navigation

Problem: Navigation controls are small and shift position between screens.

Solution: Introduce a persistent, sticky footer with full-width "Next" buttons. This anchors the user experience and provides a clear, reachable hit target on all device sizes.

Date Picker

Problem: The default calendar widget requires excessive tapping to select a birth year.

Solution: Replace with a native iOS/Android scroll-wheel picker (Spinner). This allows rapid year selection and aligns with platform-specific interaction patterns.

Location Selection

Problem: Relying solely on map-pin dragging can be imprecise.

Solution: Integrate the Google Places Autocomplete API. This offers a "Search-first" experience, allowing users to select verified locations instantly with zero ambiguity.

Design System Consistency

Problem: Corner radii and element spacing are inconsistent throughout the app.

Solution: Implement a strict "Design Token" system. By defining global variables for radius, spacing, and color, we ensure every screen feels part of a cohesive, premium product.
